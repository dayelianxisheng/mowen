### 1.2 关键概念速览

```
ROS2 项目的基本单元是"包"（package）
每个包 = package.xml(声明依赖) + CMakeLists.txt(编译规则) + 实际文件

launch/    → 启动脚本，一键启动多个节点
config/    → 算法参数（Cartographer 用 .lua，Nav2 用 .yaml）
urdf/      → 机器人运动学模型（连杆、关节、质量）
models/    → Gazebo 仿真模型（SDF 格式，含物理插件）
worlds/    → 仿真世界描述（地面、障碍物、光照）
rviz/      → 可视化界面布局文件
```

### 1.3 为什么要用 Docker

你的宿主机装的是 **ROS2 Jazzy + Gazebo Harmonic**，而 turtlebot3 项目是 **ROS2 Humble + 经典 Gazebo (Gazebo 11)**。两者的仿真体系完全不同：

|                    | Humble (Docker容器内)         | Jazzy (宿主机)            |
| ------------------ | ----------------------------- | ------------------------- |
| Gazebo             | Gazebo 11 经典版              | Gazebo Harmonic (gz-sim8) |
| 仿真启动           | `gazebo_ros` 包               | `ros_gz_sim` 包           |
| 驱动插件           | `libgazebo_ros_diff_drive.so` | `gz_ros2_control`         |
| 与 turtlebot3 兼容 | ✅ 完全兼容                    | ❌ 需要全部重写            |

**结论：Docker 容器是最快路径。**

## 3. Docker 环境搭建

### 3.1 前置条件

```bash
# 确认 Docker 已安装
docker --version

# 确认 NVIDIA GPU 支持
docker run --rm --gpus all nvidia/cuda:12.6.3-base-ubuntu24.04 nvidia-smi

# X11 转发（让 Docker 里的 GUI 能显示到宿主机屏幕）
xhost +local:docker
```

### 3.2 Dockerfile

**文件：`docker/Dockerfile`**

```dockerfile
FROM robotis/turtlebot3:humble-latest

# 工作空间目录
ENV MOWEN2_WS=/root/mowen2_ws
RUN mkdir -p ${MOWEN2_WS}/src

# 写 bashrc，进入容器自动 source 环境
RUN echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc && \
    echo "source \${MOWEN2_WS}/install/setup.bash" >> ~/.bashrc && \
    echo "export ROS_DOMAIN_ID=30" >> ~/.bashrc && \
    echo "export GAZEBO_MODEL_PATH=\${GAZEBO_MODEL_PATH}:\${MOWEN2_WS}/install/mowen2_gazebo/share/mowen2_gazebo/models" >> ~/.bashrc && \
    echo 'alias cb="colcon build --symlink-install"' >> ~/.bashrc

WORKDIR ${MOWEN2_WS}
CMD ["bash"]
```

**关键点说明：**

- `FROM robotis/turtlebot3:humble-latest` — 这个镜像已经包含 ROS2 Humble + Cartographer + Nav2 + Gazebo，不需要额外安装
- `GAZEBO_MODEL_PATH` — 告诉 Gazebo 去哪找你的机器人模型文件
- `alias cb` — 快速编译命令

### 3.3 docker-compose.yml

**文件：`docker/docker-compose.yml`**

```yaml
services:
  mowen:
    container_name: mowen_sim
    build:
      context: .
      dockerfile: Dockerfile
    tty: true
    restart: unless-stopped
    cap_add:
      - SYS_NICE
    ulimits:
      rtprio: 99
      rttime: -1
      memlock: 8428281856
    network_mode: host            # 宿主机网络（ROS2 通信需要）
    ipc: host
    pid: host
    environment:
      - DISPLAY=${DISPLAY}
      - QT_X11_NO_MITSHM=1
    volumes:
      - /dev:/dev
      - /dev/shm:/dev/shm
      - /run/udev:/run/udev
      - /tmp/.X11-unix:/tmp/.X11-unix:rw       # X11 套接字
      - /tmp/.docker.xauth:/tmp/.docker.xauth:rw
      - ./workspace:/workspace
      - ../src:/root/mowen2_ws/src              # ★ 源码挂载，宿主机改代码容器内实时同步
    privileged: true
    command: bash
```

**关键点说明：**

- `../src:/root/mowen2_ws/src` — 把宿主机的 `src/` 挂载到容器内，宿主机改代码容器内即刻生效，不需要重新拷贝
- `network_mode: host` — ROS2 的 DDS 发现机制需要，不然容器内外节点互相发现不了

### 3.4 container.sh 容器管理脚本

**文件：`docker/container.sh`**

```bash
#!/bin/bash
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
CONTAINER_NAME="mowen_sim"

show_help() {
    echo "Usage: $0 [start|enter|stop]"
}

start_container() {
    if [ -n "$DISPLAY" ]; then
        xhost +local:docker || true
    fi
    echo "Starting Mowen2 container..."
    docker compose -f "${SCRIPT_DIR}/docker-compose.yml" pull
    docker compose -f "${SCRIPT_DIR}/docker-compose.yml" up -d
}

enter_container() {
    if ! docker ps | grep -q "$CONTAINER_NAME"; then
        echo "Error: Container is not running. Run '$0 start' first."
        exit 1
    fi
    docker exec -it "$CONTAINER_NAME" bash
}

stop_container() {
    if ! docker ps | grep -q "$CONTAINER_NAME"; then
        echo "Container is not running."
        exit 1
    fi
    docker compose -f "${SCRIPT_DIR}/docker-compose.yml" down
}

case "$1" in
    help)  show_help ;;
    start) start_container ;;
    enter) enter_container ;;
    stop)  stop_container ;;
    *)     show_help ;;
esac
```

**使用方法：**

```bash
cd /home/qc/resource/code/ros/edu/2_Gazebo_simulation/docker
chmod +x container.sh

./container.sh start    # 启动容器
./container.sh enter    # 进入容器
./container.sh stop     # 停止容器
```

## 4. 仿真环境搭建 (mowen2_gazebo)

这是整个项目的核心包，负责在 Gazebo 中启动仿真世界并放入你的 mowen2 机器人。

### 4.1 package.xml — 包声明

**文件：`src/mowen2_gazebo/package.xml`**

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>mowen2_gazebo</name>
  <version>1.0.0</version>
  <description>Gazebo simulation package for Mowen2 mecanum robot</description>
  <maintainer email="your@email.com">Your Name</maintainer>
  <license>BSD</license>

  <buildtool_depend>ament_cmake</buildtool_depend>

  <depend>gazebo_ros_pkgs</depend>
  <depend>geometry_msgs</depend>
  <depend>nav_msgs</depend>
  <depend>rclcpp</depend>
  <depend>sensor_msgs</depend>
  <depend>tf2</depend>

  <export>
    <build_type>ament_cmake</build_type>
    <!-- ★ 关键：告诉 Gazebo 去哪找模型 -->
    <gazebo_ros gazebo_model_path="${prefix}/models"/>
  </export>
</package>
```

**关键点说明：**

- `<gazebo_ros gazebo_model_path="${prefix}/models"/>` — 这一行让 Gazebo 能通过 `model://mowen2` 这种 URI 找到你的机器人模型。必须在 export 里声明。

### 4.2 CMakeLists.txt — 编译规则

**文件：`src/mowen2_gazebo/CMakeLists.txt`**

```cmake
cmake_minimum_required(VERSION 3.5)
project(mowen2_gazebo)

if(NOT CMAKE_CXX_STANDARD)
  set(CMAKE_CXX_STANDARD 17)
endif()

find_package(ament_cmake REQUIRED)

# 把所有目录安装到 share/mowen2_gazebo/
install(DIRECTORY launch models rviz urdf worlds
  DESTINATION share/${PROJECT_NAME}
)

ament_package()
```

**关键点说明：**

- 这个包不编译 C++ 代码（所有插件都是系统已有的 .so 文件），所以只需要把目录安装到正确位置
- `install(DIRECTORY ...)` — colcon 会把 listed 的目录复制到 `install/mowen2_gazebo/share/mowen2_gazebo/` 下

#### 核心参数说明

**物理参数：**

- `mass=0.486` — 底盘质量 ~0.5kg，来自原 xacro 文件
- 轮子质量 ~0.05kg/个（4个共 ~0.2kg）
- 惯性矩阵来自原 xacro 文件

**坐标系：**

| 位置            | 值             | 说明                         |
| --------------- | -------------- | ---------------------------- |
| 轮子 Y 方向间距 | ±0.1454m       | 总车宽 ~0.29m                |
| 前轮 X          | +0.127m        | 在底盘中心前方               |
| 后轮 X          | -0.080m        | 在底盘中心后方（前后不对称） |
| 激光雷达高度    | z=0.136m       | 约在底盘顶部                 |
| 相机高度        | z=0.200m       | 最高点                       |
| IMU 安装        | 180度绕X轴翻转 | 可能是实际安装方向           |

**插件：**

- `libgazebo_ros_planar_move.so` — 平面移动（不需要差速/麦克纳姆轮运动学）
- `libgazebo_ros_imu_sensor.so` — 200Hz IMU，0.0002rad/s 角速度噪声
- `libgazebo_ros_ray_sensor.so` — 10Hz 360度激光，0.12-12m范围，1cm噪声
- `libgazebo_ros_joint_state_publisher.so` — 发布4个轮子关节状态

#### 4.9.3 empty_world.launch.py

**文件：`src/mowen2_gazebo/launch/empty_world.launch.py`**

```python
import os
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch.substitutions import LaunchConfiguration

def generate_launch_description():
    launch_file_dir = os.path.join(
        get_package_share_directory('mowen2_gazebo'), 'launch')
    pkg_gazebo_ros = get_package_share_directory('gazebo_ros')

    use_sim_time = LaunchConfiguration('use_sim_time', default='true')
    x_pose = LaunchConfiguration('x_pose', default='0.0')
    y_pose = LaunchConfiguration('y_pose', default='0.0')

    world = os.path.join(
        get_package_share_directory('mowen2_gazebo'),
        'worlds',
        'empty_world.world'
    )

    # 1. Gazebo 服务端（物理引擎）
    gzserver_cmd = IncludeLaunchDescription(
        PythonLaunchDescriptionSource(
            os.path.join(pkg_gazebo_ros, 'launch', 'gzserver.launch.py')
        ),
        launch_arguments={'world': world}.items()
    )

    # 2. Gazebo 客户端（3D渲染窗口）
    gzclient_cmd = IncludeLaunchDescription(
        PythonLaunchDescriptionSource(
            os.path.join(pkg_gazebo_ros, 'launch', 'gzclient.launch.py')
        )
    )

    # 3. Robot State Publisher（发布 TF 树）
    robot_state_publisher_cmd = IncludeLaunchDescription(
        PythonLaunchDescriptionSource(
            os.path.join(launch_file_dir, 'robot_state_publisher.launch.py')
        ),
        launch_arguments={'use_sim_time': use_sim_time}.items()
    )

    # 4. 生成机器人
    spawn_cmd = IncludeLaunchDescription(
        PythonLaunchDescriptionSource(
            os.path.join(launch_file_dir, 'spawn_mowen2.launch.py')
        ),
        launch_arguments={
            'x_pose': x_pose,
            'y_pose': y_pose
        }.items()
    )

    ld = LaunchDescription()
    ld.add_action(gzserver_cmd)
    ld.add_action(gzclient_cmd)
    ld.add_action(robot_state_publisher_cmd)
    ld.add_action(spawn_cmd)
    return ld
```

**关键点说明：**

- 4 个 IncludeLaunchDescription 分别启动物理引擎、渲染界面、TF发布、机器人生成
- 启动顺序由 ROS2 launch 系统自动按依赖关系编排
- `Gazebo` 启动需要设置 `GAZEBO_PLUGIN_PATH` 环境变量（已在 Docker 镜像中设置）
